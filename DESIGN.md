# Polyglow Design

## 定位

Polyglow 是 Astro 优先的静态多语言博客主题。设计目标是让文章、分类、标签、作者和搜索内容清晰可读，同时保留轻量的玻璃拟态、图片动态取色和封面卡片识别度。

本文只约束视觉、交互和组件实现。产品范围、工程原则和验收流程以 `AGENTS.md`、`.specify/memory/constitution.md` 和 `specs/` 为准。

## 基础原则

- 公开页面优先使用 Astro 组件、Tailwind CSS 和静态 HTML。
- 不为展示型 UI 引入 React、shadcn 或其他运行时组件库。
- 客户端脚本只服务主题切换、语言切换、移动导航、Pagefind 搜索、动态玻璃取色、Astro View Transitions 和可选统计。
- 默认视觉基础使用 Tailwind neutral、语义 CSS 变量和用户设备字体栈。
- 可采用者修改的内容必须集中在配置、内容集合、样式 token 和文档说明中。

## 品牌体验

- 视觉关键词：内容优先、克制、清透、图像驱动、阅读友好。
- 页面第一屏必须直接呈现站点内容或内容导航，不做营销式空白 hero。
- 主题默认不使用强品牌色。图片色彩可以影响卡片玻璃层，但不得污染全站基础色。
- 品牌名统一使用 `Polyglow`。

## 色彩

- 基础色使用 neutral 明暗两套语义 token：`background`、`foreground`、`card`、`muted`、`accent`、`border`、`ring`。
- 组件样式引用语义 token 或 Tailwind neutral 工具类，不直接散落临时 hex 色值。
- 明暗主题必须同时检查文字、边框、玻璃层、悬停态和焦点态。
- 动态取色只用于封面卡片、文章 hero 等图像相邻区域，必须保留足够文字对比度。

## 字体与排版

- 字体栈使用系统字体，不默认加载远程字体。
- 正文使用 Tailwind CSS Typography，保持合理行宽和段落节奏。
- 英语等西文正文左侧排列。
- 中文、日文、韩文正文使用两端排列，并通过 `text-autospace` 改善 CJK 与拉丁字母、数字混排间距。
- 阿拉伯语等 RTL 语言必须通过 `dir` 和 locale 配置控制方向，不用页面级临时覆盖。
- 标题层级必须反映内容结构，不用超大标题填充普通列表页或设置页。

## 布局

- 桌面端使用内容宽度约束和清晰栅格，移动端保持单列优先。
- 首页和文章列表页的分页数量为每页 20 篇；分类页和标签页的分页数量为每页 12 篇。
- Header 和 Footer 保持轻量、稳定、可扫描，不承担正文解释功能。
- 页面区块保持全宽背景或无框布局；卡片只用于文章、作者、搜索结果、错误恢复等具体内容单元。
- 不在卡片内部再嵌套装饰性卡片。

## 卡片系统

- 封面卡片以图片为主体，使用玻璃内容层承载标题、分类、日期和摘要。
- 文本卡片用于归档、搜索结果、作者附属内容等低图像依赖场景。
- 卡片圆角、阴影、玻璃边框和 hover 动效来自统一样式，不为单个页面添加一次性视觉特例。
- 图片必须通过 `OptimizedPicture` 或 Astro 图片管线输出，保留 `alt`、宽高和懒加载策略。
- hover 可以增强图片缩放和玻璃层反馈，但必须尊重 `prefers-reduced-motion`。

## 组件规则

- 图标统一通过 `src/components/icons/Icon.astro`、`astro-icon` 和 `@iconify-json/lucide` 使用 Lucide 图标。
- 按钮和可点击元素必须有清晰 hover、focus-visible、active 和 disabled 状态。
- 主题切换、语言切换、移动菜单和分页必须可键盘操作。
- 搜索页只加载 Pagefind 相关脚本和样式，普通内容页不加载搜索资产。
- 文章页 hero、prose、标签、作者信息和相邻文章导航必须在移动端保持可读。

## 可访问性

- 目标为 WCAG 2.2 AA。
- 所有交互控件必须具备可见焦点状态。
- 图标按钮必须提供可理解的文本标签或 `aria-label`。
- 图片装饰性使用空 `alt`，内容性图片必须提供准确 `alt`。
- 颜色、玻璃层和图片蒙层不得作为传达状态的唯一方式。
- 页面在无 JavaScript 时仍能阅读核心内容和访问主要链接。

## 性能

- 不引入影响主题基线的运行时 UI 框架。
- 不在全站加载只属于搜索、统计或特定交互的脚本。
- 图片优先输出现代格式并设置稳定尺寸，避免布局跳动。
- View Transitions 只用于轻量页面过渡，不增加内容页交互负担。
- 外部服务保持可选；没有环境变量或凭据时，本地开发和静态构建必须可用。

## 禁止项

- 不把 React、shadcn 或 TypeUI 设计技能包作为主题基线依赖。
- 不复制外部设计技能中的品牌风格到 Polyglow。
- 不使用单一强色系覆盖全站 neutral 基础。
- 不使用装饰性渐变球、复杂背景噪声或与内容无关的大面积视觉素材。
- 不为单个页面写难以复用的颜色、间距、圆角和阴影规则。
- 不让玻璃蒙层降低标题、摘要、分类和日期的可读性。

## 质量检查

修改 UI、布局、样式或交互后，至少检查：

- 桌面、平板、手机视口无文字重叠和横向滚动。
- 明暗主题下 Header、Footer、卡片、prose、分页和搜索控件可读。
- 语言切换、主题切换、移动菜单、分页和搜索可键盘操作。
- RTL 页面方向正确；中日韩正文和西文正文排列符合 locale 规则。
- `pnpm lint`、`pnpm check` 和相关 Playwright 用例通过。
